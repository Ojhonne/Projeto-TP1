31/*
32Utilizei o como parametro dos setters o uso da referencia
33dessa maneira não é criado copia alguma e é ultilizado a referencia daquele objeto
34o uso const antes impede do metodo alterar o valor da referencia
35
36Utilizei o const Tipo& nos getters pela mesma razao anterior
37Retornar uma referência ao atributo original ao inves de retornar uma copia
38o const impede de quem receber essa referencia, alterar o valor presente nela
39isso mantem o encapsulamento.
40
41Essa maneiras contribui com a perforamance do programa
42devolver uma referência é tão rápido e leve quanto
43devolver um número inteiro simples (um ponteiro)
44*/
