Descrição detalhada

Entidade que representa uma História de Usuário no sistema.

Uma História de Usuário agrupa as informações de um requisito ágil, compondo-se de múltiplos objetos de domínio: (Código, Título, Papel, Ação, Valor, Estimativa, Prioridade e Estado).
Cada atributo é reponsável por validar seu formato e regra.
Os dominios utilizados são:

  • Codigo
  • Texto
  • Tempo
  • Prioridade
  • Estado
Observação
Decisão de Arquitetura e Performance: Todos os métodos de acesso (getters) e modificadores (setters) desta classe utilizam passagem e retorno por referência constante (const Tipo&). Isso garante alta performance ao evitar chamadas aos construtores de cópia dos domínios, além de preservar rigorosamente o encapsulamento da entidade.
