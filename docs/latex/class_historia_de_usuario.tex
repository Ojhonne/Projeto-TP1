\doxysubsection{Detailed Description}
Entidade que representa uma História de Usuário no sistema. 

Uma História de Usuário agrupa as informações de um requisito ágil, compondo-\/se de múltiplos objetos de domínio:\+ (Código, Título, \doxylink{class_papel}{Papel}, Ação, Valor, Estimativa, \doxylink{class_prioridade}{Prioridade} e \doxylink{class_estado}{Estado}). ~\newline
Cada atributo é reponsável por validar seu formato e regra. ~\newline
Os dominios utilizados são:\+
\begin{DoxyItemize}
\item \doxylink{class_codigo}{Codigo}
\item \doxylink{class_texto}{Texto}
\item \doxylink{class_tempo}{Tempo}
\item \doxylink{class_prioridade}{Prioridade}
\item \doxylink{class_estado}{Estado}
\end{DoxyItemize}

\begin{DoxyNote}{Note}
{\bfseries{Decisão de Arquitetura e Performance:\+}} Todos os métodos de acesso (getters) e modificadores (setters) desta classe utilizam passagem e retorno por referência constante ({\ttfamily const Tipo\&}). Isso garante alta performance ao evitar chamadas aos construtores de cópia dos domínios, além de preservar rigorosamente o encapsulamento da entidade. 
\end{DoxyNote}
